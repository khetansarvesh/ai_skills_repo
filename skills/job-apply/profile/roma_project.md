`Executor Node Solution`

Turn executor node into parallel tool calling agent instead of react / codeact.

- Earlier we kept it ReAct cause DsPy gives ReAct and CodeAct out of box but does not gives parallel tool calling out of the box.
- Moreover earlier we did not support using different models for different agents in ROMA hence we could not use a good tool calling agent for the executor node and hence a looping agent was important so that tool calling occured eventually if it failed in initial interations of ReAct / CodeAct.
- Hence I made the executor node a parallel tool calling node. Thought for 90% of the queries there was just 1 tool call occuring per executor node since the task was already decomposed using the ROMA Framework.
